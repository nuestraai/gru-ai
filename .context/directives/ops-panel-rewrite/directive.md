# OpsPanel Rewrite — System Overview Panel

## CEO Brief

The current OpsPanel is built around the old categories/features/backlogs data model. Categories have been killed — replaced by the `category` field on directives. The panel needs a full rewrite.

Key direction from CEO:
- "I don't think it should be called ops" — rename the panel
- "categories already killed" — remove category-centric UI
- "this should serve as an overview of the system" — make it a system overview
- "lets think what should be there and what's the design" — needs brainstorming on content and design

The panel should answer: **"What is the state of the system right now?"** — at a glance, for the CEO.
